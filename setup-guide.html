1

สร้าง Google Sheets และ Deploy Apps Script

ประมาณ 10-15 นาที

  1. 1
    สร้าง Google Spreadsheet ใหม่

    ไปที่ sheets.new หรือ Google Drive → New → Google Sheets
    ตั้งชื่อไฟล์: ระบบใบลา APPLEPINE DIGITAL

  2. 2
    เปิด Apps Script

    ใน Google Sheets → เมนู Extensions → Apps Script
    หน้าต่าง Apps Script จะเปิดขึ้นมา

  3. 3
    วางโค้ดจากไฟล์ Code.gs

    ลบโค้ดเดิมทั้งหมด แล้ว copy โค้ดจากไฟล์ Code.gs มาวางทั้งหมด
    กด Ctrl+S หรือ ⌘+S เพื่อบันทึก

  4. 4
    Deploy เป็น Web App

    กดปุ่ม Deploy (มุมขวาบน) → New Deployment
    ตั้งค่าดังนี้:

    # ตั้งค่า Deployment: Type : Web App Description : Leave System v1.0 Execute as : Me (your email) Who has access: Anyone ← สำคัญมาก!

    กด Deploy → อนุญาต Permissions ที่ขอ → คัดลอก URL ที่ได้

  5. 5
    คัดลอก Web App URL

    URL จะมีหน้าตาแบบนี้:

    https://script.google.com/macros/s/AKfycby...XXXXX/exec

    📌 เก็บ URL นี้ไว้ใช้ในขั้นตอนถัดไป

💡 ถ้า Deploy ครั้งแรกแล้วต้องแก้โค้ด ให้ Deploy ใหม่เป็น "New Deployment" ไม่ใช่ "Manage Deployments"
2

แก้ไข index.html ใส่ Apps Script URL

ประมาณ 2 นาที

  1. 1
    เปิดไฟล์ index.html ด้วย text editor

    ใช้ VS Code, Notepad++, หรือ TextEdit ก็ได้

  2. 2
    หาบรรทัด APPS_SCRIPT_URL

    ค้นหาคำว่า YOUR_APPS_SCRIPT_URL_HERE และแทนที่ด้วย URL จากขั้นตอนที่ 1

    // ก่อนแก้ไข: APPS_SCRIPT_URL: 'YOUR_APPS_SCRIPT_URL_HERE', // หลังแก้ไข: APPS_SCRIPT_URL: 'https://script.google.com/macros/s/AKfycb.../exec',
  3. 3
    บันทึกไฟล์

    กด Ctrl+S บันทึก — แถบเตือนสีส้มบนหน้าเว็บจะหายไปเมื่อ URL ถูกต้อง

⚠️ อย่าแชร์ URL ของ Apps Script ต่อสาธารณะ เพราะใครก็ submit ข้อมูลเข้า Sheets ได้
3

สร้าง GitHub Repo + เปิด GitHub Pages

ประมาณ 5-10 นาที

  1. 1
    สร้าง Repository ใหม่บน GitHub

    ไปที่ github.com/new แล้วตั้งค่า:

    Repository name : apd-leave-system Visibility : Public ← จำเป็นสำหรับ GitHub Pages (free plan) Add README : ✓ เลือกไว้ก็ได้
  2. 2
    Upload ไฟล์ขึ้น Repository

    วิธีที่ง่ายที่สุด — ลาก & วางในเบราว์เซอร์:

    เปิด repo → กด Add file → Upload files
    ลากไฟล์ index.html และ setup-guide.html ขึ้นไป
    เขียน commit message: Add leave system v1.0
    กด Commit changes

  3. 3
    เปิดใช้งาน GitHub Pages

    ใน repo → Settings → เลื่อนลงหา Pages (sidebar ซ้าย)
    ตั้งค่า Source:

    Source : Deploy from a branch Branch : main → / (root) กด Save
  4. 4
    รอ 1-3 นาที แล้วเปิด URL

    GitHub Pages URL จะเป็นแบบนี้:

    https://username.github.io/apd-leave-system/

    เปลี่ยน username เป็น GitHub username ของ Joe

🎉 เสร็จสิ้น!

ระบบใบลาพร้อมใช้งานแล้ว · ส่ง URL ให้ทีมได้เลย
ข้อมูลจะถูกเก็บใน Google Sheets โดยอัตโนมัติ

💡 ทุกครั้งที่แก้ไข index.html ให้ upload ไฟล์ใหม่ขึ้น GitHub อีกครั้ง GitHub Pages จะอัปเดตภายใน 1-2 นาที

ข้อมูลที่เก็บใน Google Sheets

ทุก submission จะสร้าง row ใหม่ใน Sheets พร้อม column ดังนี้:

เลขที่อ้างอิง | วันที่-เวลาส่ง | รหัสพนักงาน | ชื่อ-นามสกุล | แผนก | ตำแหน่ง อีเมล | เบอร์โทร | ประเภทการลา | วันที่เริ่มลา | วันที่กลับมา | จำนวนวัน เหตุผล | ผู้อนุมัติ | ผู้รับผิดชอบแทน | สถานะ | หมายเหตุ HR

HR สามารถอัปเดตคอลัมน์ สถานะ เป็น อนุมัติ หรือ ไม่อนุมัติ ได้โดยตรงใน Sheets
